# Project

`flyto-blog` is Flyto2's public educational content site.

Primary jobs:

- Capture SEO/AEO/GEO demand for automation, CTEM, attack surface, dark web,
  AI security, pentest, and workflow topics.
- Explain concepts in a way AI answer engines can summarize and cite.
- Route readers to landing pages for product positioning and docs for technical
  mechanics.
- Provide reviewable source material for social posts, videos, feeds, package
  pages, community discussions, and answer-engine citations.

Content must support all five Flyto2 product lines:

- Cloud / Apps / Automation
- Security
- Data
- Zero-person Company Agent
- Big Data / Intelligence

The Blog owns education and discovery, not product availability, billing,
deployment authority, social credentials, video upload authority, customer
evidence, or the normative API contract. Claims about shipped features and
counts must resolve to current Flyto2 source, Docs, or product pages.

Release completion requires every executable helper, article, workflow,
configuration input, distribution plan, and public asset to remain assigned in
`docs/documentation-manifest.json` and discoverable through
`docs/reference/`.
